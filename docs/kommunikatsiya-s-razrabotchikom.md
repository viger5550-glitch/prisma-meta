# Как общаться по задачам сайта и домена

Документ для проекта PRISMA Meta: договорённости, чтобы объяснения были **понятны без инженерного жаргона**. Его можно показывать разработчику или ассистенту и просить следовать этим правилам.

## Зачем это нужно

Фразы вроде «крутится на Railway», «повесить на корень», «apex», «DNS зона» без расшифровки часто **ничего не объясняют** человеку, который не занимается серверами каждый день. Здесь зафиксировано: **как говорить проще** и **что заменить**.

## Правила для того, кто объясняет

1. **Сначала смысл — потом термин.** Например: «Нужно сказать интернету, по какому адресу открывать сайт (это настраивается в DNS у регистратора). Слово DNS — это как справочник: домен → куда вести людей.»
2. **Жаргон в скобках или с заменой.** Один раз: «корень домена (адрес без www, например `prisma-meta.ru`)» — дальше можно говорить «адрес без www».
3. **Не использовать разговорный IT-сленг вместо нормальных слов.** Ниже — таблица замен.
4. **Один шаг за раз.** Сначала «что сделать», потом «куда нажать», без смешивания трёх сервисов в одном абзаце.
5. **Если без термина никак** — дать **одно предложение** «что это значит для тебя» (например: «SSL — замочек в браузере, соединение шифруется»).
6. **Не считать очевидным:** «все знают», что такое DNS, деплой, репозиторий. При необходимости — одна строка объяснения.

## Словарь: вместо жаргона

| Избегать (или пояснять сразу) | Говорить так |
|-------------------------------|----------------|
| крутится, живёт на сервере | сайт **работает** на хостинге; сайт **доступен по адресу** … |
| деплоить, задеплоить | **выложить** сайт; **обновить** сайт на сервере |
| повесить запись | **добавить правило** в настройках домена |
| корень / apex / `@` | **адрес без www**, например `prisma-meta.ru`; в панели часто буква **`@`** |
| зона DNS | **список правил** для домена (куда вести почту, сайт и т.д.) |
| резолвится | **открывается по адресу**; «браузер находит нужный сервер» |
| редирект | **переброс**: набрали один адрес — открывается другой |
| SSL / сертификат | **защищённое соединение**; **замочек** в браузере |
| NS / nameservers | **серверы имён**: кто хранит правила для домена |
| CNAME / A / TXT | не начинать с букв; «Railway просит **добавить две строки** в настройках домена — я пришлю **тип** и **значение**» |
| прод / staging | **боевой сайт** для людей / **черновая копия** для проверки |

При необходимости дополняй таблицу под ваш проект.

## Как просить объяснить, если снова «непонятный язык»

Можно ответить так:

- «Объясни **без аббревиатур** и **одним сценарием**: что я делаю руками и что вижу в итоге в браузере.»
- «Замени слово X на обычное объяснение в одном предложении.»
- «Нужны только шаги: **куда зайти → что нажать → что вписать**.»
