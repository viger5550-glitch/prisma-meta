# Frontend Sprints Plan (No Implementation Yet)

## Цель

Разбить реализацию Prisma frontend на короткие, проверяемые спринты.
Этот документ плановый: без внесения кода.

## Sprint 0 — Подготовка (1-2 дня)

Фокус:
- заморозить требования по UI и архитектуре;
- зафиксировать контракт токенов;
- подготовить критерии приемки.

Deliverables:
- подтвержденные `frontend_spec.md`, `ui-tokens-spec.md`, `prisma-architecture-spec.md`;
- чеклист QA для навигации и отсутствия тупиков.

Exit criteria:
- команда согласовала список компонентов и первый релизный scope.

## Sprint 1 — Foundations (дизайн-токены + layout)

Фокус:
- внедрить единый источник токенов;
- собрать базовый каркас страниц.

Scope:
- token source;
- `Layout`;
- `Header`;
- базовые `Section`-паттерны и контейнеры.

Acceptance:
- единые отступы/типографика/цвета на всех страницах;
- логотип стабильно ведет на `/`.

## Sprint 2 — Core Components

Фокус:
- переиспользуемые компоненты для мета и лендингов.

Scope:
- `Button` (primary/secondary/states);
- `Card` (product card + states + navigation behavior);
- унификация CTA-паттернов.

Acceptance:
- карточки и кнопки работают единообразно;
- нет хардкод-стилей вне токенов для core-компонентов.

## Sprint 3 — Мета-страница

Фокус:
- собрать главный экран Prisma как системный хаб.

Scope:
- структура: `Header` + `Hero` + орбитальный блок карточек;
- адаптив: desktop orbit / mobile list.

Acceptance:
- каждая карточка ведет на корректный лендинг;
- mobile-версия читаема и функциональна без орбитальной схемы.

## Sprint 4 — Шаблон лендингов продуктов

Фокус:
- собрать единый шаблон для всех продуктовых лендингов.

Scope:
- блоки: Hero, Для кого, Что внутри, Формат/процесс, Результат, Цена, CTA;
- блок связанных продуктов и возврат на главную.

Acceptance:
- лендинг автономен как входная точка;
- лендинг встроен в экосистему (есть переходы на другие продукты и на `/`).

## Sprint 5 — CTA + ЮKassa MVP flow

Фокус:
- запустить простой платежный и пост-платежный путь.

Scope:
- `Оплатить` -> create payment -> redirect to ЮKassa;
- `success` page;
- `cancel/error` возврат на лендинг с сообщением.

Acceptance:
- полный happy path оплаты проходит end-to-end;
- при отмене/ошибке пользователь не попадает в тупик.

## Sprint 6 — Hardening и релиз

Фокус:
- полировка UX и стабильность перед релизом.

Scope:
- доступность базовых сценариев (focus, keyboard);
- финальная консистентность CTA/типографики/spacing;
- регресс-проверка маршрутов.

Acceptance:
- нет тупиковых страниц;
- навигационные и платежные сценарии проходят smoke-check.

## Сквозные правила для всех спринтов

- никакая страница не должна быть тупиком;
- на каждой странице минимум два выхода: целевой CTA + возврат в экосистему;
- единые паттерны текста и кнопок по всему сайту;
- не расширять scope спринта без явного решения.
