# Доработка домена (prisma-meta.ru)

Беклог: закрыть приведение публичного адреса сайта к нужному виду и стабильной DNS-конфигурации.

## Что произошло (ситуация на момент фиксации)

- Домен **`prisma-meta.ru`** куплен у **Рег.ру**, сайт размещён на **Railway**.
- Для привязки домена Railway даёт **две записи на каждый вариант имени**: проверочную **TXT** и привязку **CNAME** к адресу вида `*.up.railway.app`.
- В панели **Рег.ру** добавление **CNAME на «голый» домен** (в интерфейсе часто обозначается **`@`**) **не всегда сохраняется**: ошибка вида «CNAME нельзя добавить, если для этого имени уже есть запись другого типа» — из‑за этого привязка **`prisma-meta.ru` без www** в Railway долго оставалась в статусе ожидания DNS.
- Для варианта **с www** проще: **CNAME** для поддомена **`www`** обычно проходит. Сначала в DNS оказалась проверка **только** для голого домена (`_railway-verify`), а для **www** нужна **вторая** TXT с именем **`_railway-verify.www`** и **своим** значением из панели Railway — без неё **`www.prisma-meta.ru`** тоже не подтверждается, даже при правильном CNAME.
- В Railway одновременно фигурировали **два** кастомных домена (с www и без), у обоих — «ждём обновления DNS»; путаница из‑за разных наборов записей и лимита кастомных доменов на тарифе.

**Итог:** технически сайт доступен по служебному адресу Railway; **публичный «красивый» адрес** требует довести DNS до конца. Временно рабочим вариантом проще сделать **www**, после добавления **обеих** записей для www (TXT + CNAME) по инструкции Railway.

## Текущее решение (чем пользуемся сейчас)

- Публичные ссылки и проверки — с **www**: **`https://www.prisma-meta.ru`**, после того как в Рег.ру стоят:
  - **CNAME** `www` → значение из Railway для **`www.prisma-meta.ru`**;
  - **TXT** **`_railway-verify.www`** → полное значение **`railway-verify=…`** из той же инструкции Railway для www (не путать с проверкой для голого домена).
- Проверочная TXT для голого домена (**`_railway-verify`**) может оставаться, если в Railway всё ещё добавлен домен **`prisma-meta.ru`** без www.

## Целевая ситуация

- Основной адрес для людей — **`https://prisma-meta.ru`** (**без www** в строке браузера).
- Один понятный канонический URL в метаданных сайта и в коммуникации.
- В Railway — согласованная конфигурация доменов без «вечного ожидания DNS» из‑за ограничений регистратора.

## Решение по переходу на голый домен (зафиксировать при выполнении)

1. **DNS:** перевести управление записями на провайдера с поддержкой привязки **голого домена** к Railway (типовой вариант — **Cloudflare**: у домена в Рег.ру меняются только **серверы имён**, регистрация домена у Рег.ру сохраняется). В Cloudflare занести **точно те записи**, которые показывает Railway для **`prisma-meta.ru`** (без www): TXT проверки и привязку к `*.up.railway.app`.
2. **Railway:** оставить один основной кастомный домен **`prisma-meta.ru`** (или явно решить редирект www → без www в DNS/настройках провайдера).
3. **Код (по необходимости):** выставить базовый URL сайта для метаданных (**`metadataBase`**, переменные окружения с **`https://prisma-meta.ru`**) после того как домен открывается стабильно.

Альтернатива «только Рег.ру» для голого домена без дополнительного DNS у многих пользователей **не срабатывает** — повторять те же шаги без смены способа DNS обычего смысла не имеет.

## Сложность и трудозатраты (оценка)

| Аспект | Оценка |
|--------|--------|
| **Сложность** | Средняя для человека без опыта DNS; низкая для того, кто уже подключал домены и Cloudflare. Риск ошибки — неверная строка в DNS или два конфликтующих набора записей. |
| **Время активной работы** | Порядка **1–2 часов**: аккаунт у DNS-провайдера, смена NS у Рег.ру, копирование записей из Railway, проверка статуса в Railway. |
| **Ожидание сети** | **От 15 минут до 48 часов** после смены записей — не программирование, а распространение DNS. |
| **Изменения в коде** | Небольшие (**примерно 15–30 минут**): канонический URL в метаданных при необходимости. |
| **Стоимость** | Часто **0 ₽** за DNS на бесплатном тарифе Cloudflare; домен оплачивается у Рег.ру как сейчас. |

**Итог по трудозакладу:** не неделя разработки, а **разовая настройка DNS + короткая правка сайта**; календарно заложить **до 1 дня** с учётом ожидания DNS.
