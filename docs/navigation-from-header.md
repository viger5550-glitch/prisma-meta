# Навигация со страниц из хедера

Страницы, на которые ведут пункты шапки **AUTHOR** (`/about`) и **THE MANIFESTO** (`/meta/manifesto`), должны давать пользователю понятный способ **вернуться туда, откуда он пришёл** (челлендж, главная, другая внутренняя страница), а не только через логотип **PRISMA** на главную.

## Правило

- Вверху контента (перед основным заголовком страницы) размещается кнопка **«← назад»**.
- Реализация: компонент [`BackButton`](../src/components/meta/BackButton.tsx) — вызывает `router.back()` при наличии истории в браузере.
- Если истории нет (прямой заход по URL), используется **`fallbackHref`** — разумная «домашняя» точка для этой страницы:
  - **`/about`** → `fallbackHref="/challenge"` (типичный переход с челленджа).
  - **`/meta/manifesto`** → `fallbackHref="/"` (типичный переход с главной мета).

## Где уже сделано

| Страница            | Путь               | `fallbackHref` |
| ------------------- | ------------------ | -------------- |
| Об авторе           | `/about`           | `/challenge`   |
| Манифест            | `/meta/manifesto`  | `/`            |

При добавлении новых страниц из хедера — подключите `BackButton` по тому же принципу и обновите эту таблицу.
