# Product Concept: Prisma Meta Site

## 1. Что такое Prisma

Prisma — это публичный мета-сайт экосистемы продуктов автора.
Он не предоставляет основной пользовательский функционал, а помогает понять структуру продуктов и перейти в нужный внешний сервис.

## 2. Какую бизнес-задачу решает сайт

Сайт решает задачу навигации по экосистеме.

Когда продуктов становится много (чаты, боты, платформы, отдельные домены), пользователю сложно понять:
- что ему подходит;
- чем продукты отличаются;
- куда идти в первую очередь.

Prisma снижает этот барьер: дает единую карту продуктов и направляет трафик в целевые точки входа.

## 3. Какую ценность дает пользователю

Пользователь быстрее принимает решение и не теряется в каналах.

Ключевая ценность:
- прозрачность — видно весь набор продуктов и их роль;
- понятность — у каждого продукта есть краткое объяснение назначения;
- быстрый маршрут — понятный переход в нужный внешний продукт;
- меньше ошибок выбора — проще выбрать правильный формат (чат, бот, платформа, база знаний).

## 4. Основные принципы

- Экосистема, а не монолит.
  Prisma показывает систему связанных продуктов, которые живут на разных площадках.

- Автономность лендингов.
  Каждый продукт может иметь собственный лендинг и самостоятельный путь пользователя без зависимости от главной страницы.

- Связность.
  Несмотря на автономность, все точки входа согласованы по логике: единые описания ролей продуктов, понятные переходы между ними.

- Публичность и простота доступа.
  Сайт полностью открыт: без авторизации, личного кабинета и пользовательского состояния.

- Функция маршрутизатора.
  Основная функция — не "обслужить" пользователя внутри, а корректно направить его наружу.

## 5. Пользовательские сценарии

### Сценарий A: вход через мета-страницу Prisma

1. Пользователь попадает на Prisma.
2. Видит карту продуктов и короткие объяснения по каждому.
3. Сопоставляет свою задачу с назначением продукта.
4. Переходит по CTA в выбранный внешний продукт (Telegram, бот, платформа, отдельный домен).
5. Дальнейший пользовательский опыт происходит уже вне Prisma.

### Сценарий B: прямой заход на лендинг продукта

1. Пользователь приходит сразу на конкретный лендинг (по ссылке/рекомендации).
2. Получает описание именно этого продукта и целевое действие.
3. Может сразу перейти во внешний сервис.
4. При необходимости видит путь к другим продуктам экосистемы через ссылки на Prisma/связанные лендинги.
5. Prisma в этом сценарии выступает как контекстный слой, а не обязательный этап.

## 6. Почему сайт не является продуктом

Prisma не является самостоятельным продуктом, потому что:
- не содержит основного пользовательского функционала;
- не удерживает пользователя внутри длительного сценария;
- не требует аккаунта, персонализации и пользовательских данных;
- не создает основной ценностный результат внутри себя.

Его роль — точка входа и система навигации между внешними продуктами, где и происходит реальная пользовательская работа.
